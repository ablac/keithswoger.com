01 / Agent infrastructure

AgentReef

AgentReef is the coordination layer I wanted when several agents were working on one codebase. Peers can discover each other, call skills, share signed memory, and leave a durable record of the work.

The server is written in Rust and uses signed requests, scoped grants, PostgreSQL row-level security, and Redis for presence. Most of the engineering work is in keeping ownership, context, and permissions clear after the work spreads out.

  • Rust API and WebSocket server
  • Ed25519-signed calls and results
  • Shared memory with tenant isolation
  • Operator dashboard and audit trail

02 / Real-time systems

AI Battle Arena

Arena is a live combat sandbox for AI bots. Authors deploy a bot, watch a match, and iterate against a scoreboard that keeps moving.

A Go server owns the match state. WebSockets stream combat to the browser spectator UI, while Python and Node SDKs give bot authors a practical way in. The public repository includes the protocol, examples, and contribution guide.

  • Go server and WebSocket protocol
  • Browser spectator interface
  • Python and Node SDKs
  • Public AGPL-licensed repository

03 / Visual computing

Aquarium

Aquarium is a cinematic underwater world that runs in the browser. It has a research station, schools of fish, ambient sound, changing light, and enough movement to make performance work matter.

The current experience runs on Babylon.js and WebGL. I also wrote the clean-room rebuild plan for a WebGPU-first version with deterministic simulation, adaptive quality, safe teardown, and accessibility built into the architecture.

  • Procedural 3D world and creature simulation
  • Babylon.js and WebGL
  • Adaptive rendering and reduced-motion planning
  • WebGPU-first rebuild specification
